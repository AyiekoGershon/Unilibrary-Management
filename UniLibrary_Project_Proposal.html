üéØ Problem Statement

University libraries face significant operational challenges in managing the high volume of student bags and personal belongings, especially during peak hours. Traditional manual tracking methods present multiple issues:

  • Time-Consuming: Manual record keeping is slow and labor-intensive
  • Error-Prone: Paper-based systems lead to lost or misplaced items
  • Inefficient: No real-time visibility of checked-in items
  • Unscalable: Difficult to manage during busy periods
  • No Accountability: Difficult to track who owns which bag

üîÑ User Workflow

  1. Authentication: Librarian logs in with Google OAuth
  2. Check-In:
    • Enter student email and bag description
    • System generates unique QR code (e.g., LIB-ABC123)
    • Email confirmation sent to student with QR code image
    • Bag status set to "checked_in"
  3. Active Management:
    • Dashboard shows all currently checked-in bags
    • Real-time updates across all sessions
    • Quick search by student or reference code
  4. Check-Out:
    • Scan QR code or select from list
    • Bag status set to "checked_out"
    • Check-out timestamp recorded + email recap (time spent & streak)

üîê Security & Compliance

  • Authentication: OAuth 2.0 via Google for secure user verification
  • Data Protection: Row-level security (RLS) in Supabase ensures users only see their data
  • Encryption: HTTPS/TLS for all data transmission
  • API Security: API keys stored in environment variables, never exposed to frontend
  • Database Security: PostgreSQL with built-in security features
  • GDPR Compliant: User data handling follows privacy regulations
  • Serverless Isolation: Edge Functions run in isolated containers
